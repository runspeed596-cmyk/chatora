## Description

This workflow defines the step-by-step operational process that the AI Agent must follow to design, build, validate, and deliver the MiniChat application with maximum quality, security, and reliability.

The Agent must strictly follow this workflow without deviation.

---

## 1. INITIALIZATION PHASE

### Objective
Prepare the system environment and understand project scope.

### Actions
- Read and analyze:
  - rules.md
  - system prompt
  - project requirements
- Validate current development phase.
- Identify active constraints.
- Confirm available resources.

### Output
- Confirmed scope
- Active phase declaration
- Risk overview

---

## 2. ANALYSIS PHASE

### Objective
Fully understand technical and business requirements.

### Actions
- Analyze legacy backend (Spring Boot).
- Identify obsolete components.
- Map required features.
- Extract dependencies.
- Detect architectural conflicts.
- Document assumptions.

### Output
- System analysis report
- Dependency map
- Risk list

---

## 3. ARCHITECTURE DESIGN PHASE

### Objective
Design a scalable and secure system.

### Actions
- Define Clean Architecture layers.
- Design database schema.
- Define API contracts.
- Design WebSocket protocols.
- Define security model.
- Select design patterns.
- Plan modularization.

### Output
- Architecture blueprint
- Database ERD
- API specification
- Security model

---

## 4. IMPLEMENTATION PHASE

### Objective
Build production-grade systems.

### Actions
- Implement backend modules.
- Implement authentication.
- Implement matching engine.
- Implement chat system.
- Implement localization.
- Implement permission system.
- Apply SOLID principles.
- Apply coding standards.

### Output
- Compilable source code
- Configured environments
- Integrated modules

---

## 5. UI/UX DEVELOPMENT PHASE

### Objective
Create a high-quality user experience.

### Actions
- Design layouts and components.
- Implement themes.
- Implement navigation.
- Add animations.
- Apply accessibility rules.
- Implement responsive behavior.

### Output
- UI component library
- Navigation system
- Theme system

---

## 6. INTEGRATION PHASE

### Objective
Connect all system parts.

### Actions
- Connect REST APIs.
- Connect WebSocket.
- Integrate WebRTC.
- Handle reconnection.
- Implement fallback logic.
- Validate data consistency.

### Output
- End-to-end connected system
- Real-time communication enabled

---

## 7. TESTING & VALIDATION PHASE

### Objective
Ensure system reliability.

### Actions
- Perform unit tests.
- Perform integration tests.
- Perform stress tests.
- Perform security tests.
- Perform usability tests.
- Validate localization.

### Output
- Test reports
- Bug list
- Stability confirmation

## 8. OPTIMIZATION PHASE

### Objective
Maximize performance and stability.

### Actions
- Profile memory usage.
- Optimize network traffic.
- Reduce latency.
- Optimize rendering.
- Optimize database queries.
- Tune thread usage.

### Output

### Objective
Eliminate vulnerabilities.

### Actions

- Review encryption.
- Enable certificate pinning.
- Apply rate limiting.
- Validate access control.
- Review permissions.

### Output
- Security audit report
- Hardened system

## 10. DOCUMENTATION PHASE

### Objective
Ensure maintainability.

### Actions
- Document APIs.
- Document architecture.
- Document deployment steps.
- Document troubleshooting.
- Update rules.md.

### Output
- Complete documentation set

## 11. REVIEW & APPROVAL PHASE

### Objective
Validate readiness.

### Actions
- Self-audit against rules.md.
- Verify phase completion.
- Generate summary report.
- Request user approval.

### Output
- Final review report
- Approval request

## 12. RELEASE PHASE

### Objective
Deliver stable product.

### Actions

- Prepare release notes.
- Package builds.

- Monitor metrics.

### Output
- Production release
- Monitoring dashboard

---

## 13. CONTINUOUS IMPROVEMENT PHASE

### Objective
Maintain long-term quality.

### Actions
- Analyze feedback.
- Track incidents.
- Improve architecture.
- Refactor weak areas.
- Update workflow.

### Output
- Improvement roadmap

## GLOBAL RULES

- Never skip phases.
- Never merge incomplete work.
- Never hide risks.
- Never downgrade security.
- Never sacrifice stability.
- Always document decisions.
- Always wait for user approval.

## FINAL DIRECTIVE

The Agent must execute this workflow sequentially.

Quality, security, and reliability are non-negotiable.
